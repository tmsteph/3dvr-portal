Deployment

Self-hosted GitHub + Vercel, the easy way

If the number-one constraint is “easy,” this combo mirrors the GitHub + Vercel workflow while staying fully self-hosted.

Core combo

  • Forgejo as your “GitHub” for repos, issues, PRs, and accounts. Host it at a URL like git.3dvr.tech.
  • Coolify (or /dev/push) as your “Vercel” for click-to-deploy from Git. Coolify is dashboard-first; /dev/push feels more Vercel-ish.

What you get

  • Push code → auto deploy.
  • Any language (Docker or buildpacks).
  • No hosted limits.
  • Runs on a cheap VPS, EC2, or local hardware.

Minimum-effort setup for 3dvr.tech

  1. Buy one $6–$12/month VPS.
  2. Install Coolify on it.
  3. Add Forgejo as another app in Coolify (or run Forgejo separately).
  4. Point DNS as shown below.

DNS layout

  • git.3dvr.tech → Forgejo
  • deploy.3dvr.tech → Coolify dashboard
  • *.apps.3dvr.tech → Coolify deployments (wildcard)

Decentralization later (optional)

When you are stable, you can add Radicle or federation for “real” decentralization. Keep it out of the first iteration so the workflow stays frictionless.
