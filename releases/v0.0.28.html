Release v0.0.28

Week of November 30, 2025

End-of-November build shipping internally now (external on Monday): release hub links stay current, CRM records now show exactly which fields will win in a merge before anything writes to Gun, and calendars support quick reschedules with steadier task summaries.

Overview

We focused this week on release readiness and everyday flow polish. The release landing page stays pinned to the latest milestone, CRM workspaces can stage merges without risking data collisions, and the calendar/task stack supports faster adjustments when plans shift. Underneath, Gun session locks and retry loops aim to keep data in sync even on flaky connections. This is an internal drop at the end of November, with external publication set for Monday.

Portal quick links

  • Portal home – hop across workspaces and the release hub.
  • Release hub – browse every weekly milestone.
  • CRM & contacts – review merge candidates before committing to Gun.
  • Calendar – reschedule entries and keep invite status visible.
  • Tasks – scan updated summaries and cross-links to calendar items.

Portal shell & navigation

  • Highlight the current weekly milestone on the release hub so contributors land on the newest notes first.
  • Reuse the consistent back-links and navigation affordances to speed up context switching between releases and active workspaces.
  • Retain dark theme spacing and typography adjustments that keep long-form notes readable on mobile and desktop.

CRM & contacts

  • Surface merge previews before writes, with a field-by-field view of what changes, so CRM updates stay in sync with shared Gun nodes.
  • Keep contact detail panels aligned with identity fields used elsewhere in the portal (tasks, meetings, and rewards), reducing duplicate typing.
  • Clarify inline guidance for contributors linking external accounts, improving how we test cross-device replication.

Calendar & tasks

  • Add reschedule prompts for calendar entries to simplify moving meetings while maintaining invite visibility.
  • Refresh task summaries and status badges so they reflect the latest schedule changes from the calendar view.
  • Preserve quick navigation between agenda items and related notes to keep workstreams connected.

Reliability & sync

  • Use Gun session locks and retry loops to keep writes queued when a connection blips, then flush changes once the node is reachable.
  • Continue testing across browsers to ensure compatibility shims, module detection, and cache-clearing steps work without manual intervention.
  • Document the sync behaviors directly in release notes so contributors can verify storage flows alongside their weekly checklists.
